# アプリでできること（MVPドラフト）

## 目的

活動ログをステータスとして定量化し、可視化して日々の変化を追えるようにする。

## MVPで実装する機能

## 1. ステータスの定量的マッピング（最小版）

まずは 2〜3 軸に絞って実装する。

- `Strength`（筋力）: 基本コンパウンド種目の重量（対体重比）から算出
- `Routine`（習慣性）: 日々のルーチン継続状況から算出
- `Health`（健康）: 睡眠時間から算出

将来的な候補（MVP外）:
- `Intelligence`（知力）
- `Skill`
- `Personnel Evaluation`

ステータスの選択機能:
- ユーザーが任意で ステータス を追加できる
- ステータスを削除できる

## 2. ステータス管理

ステータスを1~10段階で設定する

- ステータスの入力
- ステータスの編集
- ステータスの削除

## 3. ステータスの可視化

- Home 画面でレーダーチャート表示

## 4. ステータスのログ機能

時系列でステータスのログを表示できる

## 4. キャラクター画像表示（最小版）

- ステータスに応じたキャラクター画像を表示
- MVPでは生成品質最適化よりも「表示フロー成立」を優先

## MVPで必要な画面

### Home 画面

- レーダーチャート表示
- キャラクター画像表示

### ステータス画面

- 入力
- 編集
- 削除

## MVPでは実装しない（後続）

- ログイン機能
- 6軸すべての本実装
- 画像生成の高度な品質調整
- ランク / 称号などのゲーミフィケーション拡張

## Codexで進める前提

- 要件定義: 各ステータスの算出ルール（入力元・計算式・更新頻度）を固定する
- 機能開発: 上記MVP範囲（CRUD、可視化、画像表示）を優先して段階実装する
